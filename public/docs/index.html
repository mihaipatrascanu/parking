Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

Create new user

Example request:
curl --request POST \
    "http://localhost/api/user/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"name\": \"Mihai\",
    \"email\": \"mihai@mihai.com\",
    \"password\": \"password\",
    \"password_confirmation\": \"password\"
}"

const url = new URL(
    "http://localhost/api/user/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "name": "Mihai",
    "email": "mihai@mihai.com",
    "password": "password",
    "password_confirmation": "password"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request      

GET api/{parking_id}/spots

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

URL Parameters

parking_id   integer   

The ID of the parking. Example: 1

start   string   

Example: 2022-12-01T00:00:00.000Z

end   string   

Example: 2022-12-02T00:00:00.000Z

Body Parameters

datetime_range   object  optional  
datetime_range.start   string  optional  

Must be a valid date. Must be a date after or equal to now. Example: 2065-11-25

datetime_range.end   string  optional  

Must be a valid date. Must be a date after start. Example: 1987-04-24
